Introducción a Git y GitHub

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite rastrear cambios en archivos de código. Te permite trabajar en equipo, mantener un historial de cambios y revertir a versiones anteriores cuando sea necesario.

¿Qué es GitHub?

GitHub es una plataforma online que corre Git. Te permite almacenar tus repositorios en la nube, colaborar con otros desarrolladores, y compartir tu código con la comunidad.

¿Por qué usar Git y GitHub?

  • ✓ Historial completo de cambios
  • ✓ Colaboración en equipo
  • ✓ Ramificación para desarrollar features
  • ✓ Seguridad y respaldos
  • ✓ Estándar de la industria

Conceptos Básicos

📁 Repositorio (Repo)

Un espacio donde se almacena el código del proyecto junto con su historial de cambios. Puede ser local (en tu computadora) o remoto (en GitHub).

💾 Commit

Una "fotografía" de tus cambios en un momento específico. Cada commit tiene un ID único y un mensaje descriptivo de qué cambió.

$ git commit -m "Agregar función de login"

🌲 Branch (Rama)

Una copia independiente del código donde puedes trabajar sin afectar la rama principal (main/master). Perfecto para desarrollar features nuevas.

⬆️ Push

Enviar tus commits locales al repositorio remoto (GitHub).

$ git push origin main

⬇️ Pull

Traer cambios desde el repositorio remoto a tu computadora local.

$ git pull origin main

🔀 Merge (Fusión)

Combinar cambios de una rama con otra. Típicamente se usa para unir una feature branch con main.

$ git merge nombre-rama

🔗 Pull Request (PR)

Una solicitud en GitHub para revisar y fusionar cambios de una rama a otra. Es la forma estándar de colaboración.

Comandos Esenciales de Git

Configuración Inicial

$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tu@email.com"

Crear y Clonar Repositorios

$ git init (crear nuevo repo)
$ git clone https://github.com/usuario/repo.git (clonar repo)

Cambios Locales

$ git status (ver cambios)
$ git add archivo.txt (agregar archivo)
$ git add . (agregar todos)
$ git commit -m "Mensaje" (guardar cambios)

Ramas

$ git branch (listar ramas)
$ git branch nombre-rama (crear rama)
$ git checkout nombre-rama (cambiar rama)
$ git checkout -b nombre-rama (crear y cambiar)

Remoto

$ git push origin branch (enviar cambios)
$ git pull origin branch (traer cambios)
$ git merge rama (fusionar rama)

Historial

$ git log (ver histórico de commits)
$ git diff (ver diferencias)
